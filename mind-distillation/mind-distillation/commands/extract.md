# /extract - 萃取方法论

## 目的

从原始数据中萃取出结构化的方法论。

## 核心理念

**萃取的是"决策框架"，不是"知识记忆"**

重点提炼：
- 面对问题时的分析思路
- 做决策时的判断标准
- 权衡取舍时的优先级

## 执行步骤

### Step 0: 检查数据

```bash
# 检查数据目录
DATA_DIR="cases/{case}/data"
if [ ! -d "$DATA_DIR" ]; then
  echo "错误：数据目录不存在，请先运行 /crawl"
  exit 1
fi

# 统计数据量
ARTICLE_COUNT=$(ls $DATA_DIR/articles/*.md 2>/dev/null | wc -l)
QA_COUNT=$(ls $DATA_DIR/qa/*.md 2>/dev/null | wc -l)
echo "数据统计：$ARTICLE_COUNT 篇文章，$QA_COUNT 条问答"
```

### Step 1: 抽取分析样本

从数据中随机抽取样本进行分析：

- 文章：抽取5-10篇（覆盖不同领域）
- 问答：抽取10-20条（覆盖不同问题类型）

**样本选择原则**：
1. 覆盖博主涉及的主要领域
2. 包含不同长度的内容
3. 包含有明确结论的内容

### Step 2: 分析博主画像

从样本中提取：

```markdown
## 博主画像

| 维度 | 分析方法 | 输出 |
|------|----------|------|
| 领域 | 统计文章涉及的主题 | 股市、楼市、教育... |
| 背景 | 从自我介绍/引用推断 | 数据驱动的价值投资者 |
| 风格 | 分析语言特征 | 极简、务实、给判断 |
| 特点 | 提取独特观点 | 不说废话、独立决策 |
```

### Step 3: 提取决策框架

**核心方法**：从博主的判断过程中提炼原则

**分析模板**：

```markdown
## 问题：{问题}

### 博主分析过程
1. 先看什么数据/指标
2. 对比什么基准
3. 得出什么结论
4. 给什么建议

### 提炼的决策原则
- 原则1：{...}
- 原则2：{...}

### 可复用的判断标准
| 指标 | 阈值 | 判断 |
|------|------|------|
| ... | ... | ... |
```

### Step 4: 归纳核心原则

从多个样本中归纳共性：

```markdown
## 核心原则归纳

### 样本1分析
- 使用了原则A、B
- 判断依据是X、Y

### 样本2分析
- 使用了原则A、C
- 判断依据是X、Z

### 归纳结果
- 共性原则：A（出现2次）
- 判断依据：X（出现2次）

→ **提炼为核心原则A，判断依据X**
```

### Step 5: 生成方法论

使用模板生成方法论文档：

```bash
# 使用模板
TEMPLATE="skill/templates/methodology-template.md"
OUTPUT="cases/{case}/methodology.md"

# 填充内容
# 1. 博主画像
# 2. 核心决策框架
# 3. 分领域方法论
# 4. 回答风格
# 5. 风险提示与边界
```

### Step 6: 版本管理

```markdown
## 版本记录

| 版本 | 日期 | 数据量 | 更新内容 |
|------|------|--------|----------|
| v0.1 | {date} | 10条样本 | 初始框架 |
| v1.0 | {date} | 全量数据 | 完整方法论 |
```

## 萃取原则

### 1. 小样本先行
先用10条以内数据手工验证，再扩展到全量。

### 2. 从实例到抽象
先分析具体案例，再归纳一般原则，不要凭空设计。

### 3. 保留原文表达
典型表达直接引用原文，不要过度改写。

### 4. 识别边界
不仅记录博主"怎么做"，也要记录博主"不做什么"。

## 质量检查清单

- [ ] 博主画像是否准确？
- [ ] 核心原则是否有实例支撑？
- [ ] 判断标准是否具体可操作？
- [ ] 回答风格是否有典型表达示例？
- [ ] 是否记录了"不做的事"？
- [ ] 版本信息是否完整？

## 命令参数

```bash
/mind-distillation extract --case zhongyuan --sample-size 15
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| --case | 博主案例名 | 必填 |
| --sample-size | 分析样本数 | 10 |
| --output | 输出文件路径 | cases/{case}/methodology.md |
| --verbose | 显示详细分析过程 | false |

## 输出文件

`cases/{case}/methodology.md` - 完整方法论文档
