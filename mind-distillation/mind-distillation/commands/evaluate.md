# /evaluate - 方法论评估

## 目的

用 **LLM-as-Judge** 评估方法论效果，回答核心问题：
> 让AI用这个方法论回答问题，能多像博主本人？

## 执行步骤

### Step 0: 检查前置条件

```bash
# 检查方法论文件是否存在
if [ ! -f "cases/{case}/methodology.md" ]; then
  echo "错误：方法论文件不存在，请先运行 /extract"
  exit 1
fi

# 检查测试问题集
if [ ! -f "cases/{case}/data/qa-testset.json" ]; then
  echo "提示：测试问题集不存在，将从原问答中随机抽取"
fi
```

### Step 1: 准备测试问题集

从原问答数据中随机抽取5-10条作为测试集：
- 问题来自博主真实回答过的问题
- 确保覆盖方法论中涉及的主要领域
- 记录原回答作为对比基准

### Step 2: AI模拟回答

让AI阅读方法论，然后回答测试问题：

```
你现在是【博主名】，请根据以下方法论回答问题。

【方法论内容】

问题：{测试问题}

要求：
1. 遵循方法论中的决策框架
2. 模仿回答风格（极简、务实、给判断）
3. 不要编造数据或观点
```

### Step 3: LLM-as-Judge 评估

让另一个LLM对比AI回答与原回答：

```markdown
## 评估任务

对比AI回答与博主原回答，从三个维度打分（0-10分）：

### 1. 风格相似度（权重30%）
- 语言特征是否一致（极简/详细、直接/委婉）
- 信息密度是否接近
- 是否有博主的典型表达方式

### 2. 决策框架遵循（权重40%）
- 是否应用了方法论中的核心原则
- 推理逻辑是否与博主一致
- 判断结论是否相近

### 3. 实用价值（权重30%）
- 回答是否给出可操作建议
- 建议是否有参考价值
- 是否避免了空洞的泛论

## 原问题
{问题}

## 博主原回答
{原回答}

## AI模拟回答
{AI回答}

## 请输出
JSON格式：
{
  "style_score": 0-10,
  "framework_score": 0-10,
  "value_score": 0-10,
  "weighted_total": 计算加权总分,
  "feedback": "具体改进建议"
}
```

### Step 4: 生成评估报告

```markdown
# 方法论评估报告

> 评估时间：{date}
> 测试问题数：{n}条

## 总体得分

| 维度 | 得分 | 说明 |
|------|------|------|
| 风格相似度 | X/10 | {说明} |
| 决策框架遵循 | X/10 | {说明} |
| 实用价值 | X/10 | {说明} |
| **加权总分** | **X/10** | |

## 详细分析

### 表现好的方面
- ...

### 需要改进的方面
- ...

## 具体建议

1. 方法论中缺少对{领域}的决策框架
2. 回答风格需要更{特征}
3. 建议补充{具体内容}

## 与上次评估对比

| 指标 | 上次 | 本次 | 变化 |
|------|------|------|------|
| 总分 | X | Y | +Z |
```

## 评估结果用途

1. **得分 < 6分**：方法论需要大幅补充
2. **得分 6-8分**：方法论基本可用，需要微调
3. **得分 > 8分**：方法论质量良好，可投入使用

## 持续追踪

每次评估结果保存到 `cases/{case}/eval-history/`，用于追踪方法论迭代效果。

```bash
cases/{case}/eval-history/
├── 2026-03-27-eval-001.json
├── 2026-03-28-eval-002.json
└── ...
```

## 命令参数

```bash
/mind-distillation evaluate --case zhongyuan --questions 5
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| --case | 博主案例名 | 必填 |
| --questions | 测试问题数 | 5 |
| --verbose | 显示详细对比 | false |
