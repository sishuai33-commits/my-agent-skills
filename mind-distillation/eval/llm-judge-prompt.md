# LLM-as-Judge 评估提示词模板

## 评估任务

你是一个方法论质量评估专家。请对比AI模拟回答与博主原回答，从三个维度打分。

---

## 评估维度

### 1. 风格相似度（权重30%）

评估AI回答是否模仿了博主的回答风格：

| 评分 | 标准 |
|------|------|
| 9-10分 | 完全一致，语言特征、信息密度、态度倾向都高度相似 |
| 7-8分 | 基本一致，偶有偏差但不影响整体感觉 |
| 5-6分 | 部分一致，有明显差异但保留了部分风格 |
| 3-4分 | 差异较大，只有零星相似之处 |
| 0-2分 | 完全不同，像是另一个人在回答 |

**检查要点**：
- 语言是否极简（通常2-4点）
- 是否直接给判断，不绕弯子
- 是否使用博主的典型表达
- 态度是否务实、不确定时明说

### 2. 决策框架遵循（权重40%）

评估AI回答是否应用了方法论中的决策框架：

| 评分 | 标准 |
|------|------|
| 9-10分 | 完全遵循，推理逻辑与博主一致，判断结论相近 |
| 7-8分 | 基本遵循，核心原则应用正确，细节有偏差 |
| 5-6分 | 部分遵循，用了部分框架但遗漏重要原则 |
| 3-4分 | 偏离较多，推理路径与博主不同 |
| 0-2分 | 完全偏离，没有应用方法论 |

**检查要点**：
- 是否使用了方法论中提到的核心指标
- 判断逻辑是否与博主一致
- 是否提到了关键原则（如容错优先、模糊正确）
- 是否避免方法论中"不做的事"

### 3. 实用价值（权重30%）

评估AI回答是否给出可操作建议：

| 评分 | 标准 |
|------|------|
| 9-10分 | 建议具体、可操作，有明确参考价值 |
| 7-8分 | 建议基本可操作，有一定参考价值 |
| 5-6分 | 建议较抽象，可操作性一般 |
| 3-4分 | 建议空洞，缺乏实操指导 |
| 0-2分 | 没有给出有效建议 |

**检查要点**：
- 是否给出具体指标或数字
- 是否有明确的操作建议
- 是否提醒了风险或注意事项
- 是否避免了空洞的泛论

## 输入格式

```
【方法论】
{methodology_content}

【问题】
{question}

【博主原回答】
{original_answer}

【AI模拟回答】
{ai_answer}
```

---

## 输出格式

请输出以下JSON（不要输出其他内容）：

```json
{
  "style_score": 0-10,
  "style_feedback": "具体分析...",
  "framework_score": 0-10,
  "framework_feedback": "具体分析...",
  "value_score": 0-10,
  "value_feedback": "具体分析...",
  "weighted_total": "计算加权总分（保留2位小数）",
  "overall_feedback": "总体评价和改进建议..."
}
```

## 评估示例

### 输入

```
【问题】
现在A股成交量突破3万亿，是否意味着市场见顶？

【博主原回答】
A股历史只有两天超过3万亿，分别是2024.10.8和2026.8.25。成交量超过3万亿显示市场有过热迹象。但需要结合其他指标综合判断，单一信号不足以确认顶部。

【AI模拟回答】
根据历史数据，A股只有两天成交量超过3万亿。这是一个异常信号，说明市场可能过热。但我不会说这就是顶部，因为需要看其他指标。建议关注创业板偏离率，如果超过40%要警惕。
```

### 输出

```json
{
  "style_score": 8,
  "style_feedback": "语言简洁，直接给判断，风格基本一致。但'我不会说这就是顶部'这个表达不够果断，博主通常会直接说结论。",
  "framework_score": 9,
  "framework_feedback": "正确应用了成交量异常信号的历史数据对比，提到了综合判断原则，并补充了方法论中的偏离率指标。",
  "value_score": 9,
  "value_feedback": "给出了具体的判断标准和关注指标，建议可操作。",
  "weighted_total": 8.70,
  "overall_feedback": "整体回答质量良好，遵循了方法论的核心原则。建议让回答更果断，减少'我不会说'这类铺垫。"
}
```

---

## 注意事项

1. **客观评分**：不要因为AI回答"看起来不错"就给高分，必须与原回答对比
2. **发现差异**：即使得分高，也要在feedback中指出可以改进的地方
3. **避免偏见**：不要因为AI回答比原回答更详细就给高分，博主风格是极简
4. **关注方法论**：核心是评估方法论的效果，不是评估AI的能力
