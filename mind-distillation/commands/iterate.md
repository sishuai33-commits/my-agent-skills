# /iterate - 方法论迭代

## 目的

根据新数据或评估反馈，增量更新方法论。

## 触发条件

1. **新增数据**：博主发布了新内容
2. **评估反馈**：评估得分低于预期
3. **发现盲区**：用户发现方法论无法覆盖某些问题

## 执行步骤

### Step 0: 确定迭代来源

```bash
# 检查迭代触发原因
if [ -n "$NEW_DATA" ]; then
  echo "迭代来源：新增数据"
  ITERATION_TYPE="data-driven"
elif [ -f "cases/{case}/eval-report.md" ]; then
  echo "迭代来源：评估反馈"
  ITERATION_TYPE="eval-driven"
else
  echo "错误：没有明确的迭代输入"
  exit 1
fi
```

### Step 1: 数据驱动迭代（新增内容）

当博主发布新内容时：

1. **分析新内容**：识别涉及的新领域或新观点
2. **对比现有方法论**：找出方法论中缺失的部分
3. **增量补充**：只添加新的决策框架，不重写已有内容
4. **版本记录**：更新方法论版本号

```markdown
## 迭代日志

### v1.0 → v1.1 (2026-03-28)

**新增内容**：
- 新增"黄金投资逻辑"章节
- 补充"美债投资完整方法"

**修改内容**：
- 调整股市见顶信号阈值（基于最新数据）

**删除内容**：
- 无

**数据来源**：2026-03-28 新发布的3篇文章
```

### Step 2: 评估驱动迭代（质量反馈）

当评估得分不理想时：

1. **定位问题**：从评估报告中找出低分维度
2. **分析原因**：为什么方法论在这些维度表现不佳
3. **针对性改进**：

| 问题维度 | 可能原因 | 改进方向 |
|----------|----------|----------|
| 风格相似度低 | 回答模板不够具体 | 补充更多典型表达示例 |
| 决策框架遵循差 | 原则不够明确 | 添加决策流程图/判断表 |
| 实用价值低 | 建议太抽象 | 补充具体操作指南 |

### Step 3: 重新评估

迭代后必须重新运行评估，确认改进效果：

```bash
/mind-distillation evaluate --case zhongyuan
```

### Step 4: 对比分析

```markdown
## 迭代效果对比

| 指标 | 迭代前 | 迭代后 | 变化 |
|------|--------|--------|------|
| 风格相似度 | 6.5 | 7.2 | +0.7 |
| 决策框架遵循 | 7.0 | 7.8 | +0.8 |
| 实用价值 | 6.0 | 6.5 | +0.5 |
| **总分** | **6.55** | **7.25** | **+0.7** |

**结论**：迭代有效，方法论质量提升。
```

## 迭代原则

### 渐进式补充
- 只添加缺失的决策框架
- 不删除已验证有效的内容
- 保持方法论的连贯性

### 版本管理
- 每次迭代更新版本号（v1.0 → v1.1）
- 保留迭代历史记录
- 可回滚到任意版本

### 质量追踪
- 每次迭代后必须评估
- 记录得分变化趋势
- 发现迭代无效时回退

## 迭代历史格式

```markdown
# 方法论迭代历史

## v1.0 (2026-03-27)
初始版本，基于103篇文章+109条问答

## v1.1 (2026-03-28)
- 新增：黄金投资逻辑
- 修改：股市见顶信号阈值
- 评估得分：6.55 → 7.25 (+0.7)

## v1.2 (2026-04-01)
- 新增：跨境电商商业机会分析
- 补充：更多城市楼市分析方法
- 评估得分：7.25 → 7.4 (+0.15)
```

## 命令参数

```bash
/mind-distillation iterate --case zhongyuan --new-data ./new-articles.json
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| --case | 博主案例名 | 必填 |
| --new-data | 新数据路径 | 可选 |
| --from-eval | 基于最近评估迭代 | false |
